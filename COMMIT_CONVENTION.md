# Commit Convention

This project follows a simplified Conventional Commits standard.

## Format

type: short description

## Types

- chore: configuration, setup, non-functional changes
- docs: documentation
- feat: new feature
- fix: bug fix
- refactor: code improvement without behavior change
- test: test related changes

## Examples

chore: add backend project structure  
docs: update README file  
feat: implement client creation endpoint  
fix: correct JWT expiration bug  
refactor: improve database query logic
